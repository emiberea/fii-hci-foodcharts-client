Introduction

The project that we are describing in this paper is called "FoodCharts" and it is implemented for the Human-Computer Interaction course held by Dr. Sabin-Corneliu Buraga at Faculty of Computer Science, “Al.I.Cuza” University, Iasi, in the second semester of the 2015-2016 school year, and developed by the 2nd year MA students Berea Emanuel-Vasile and Serseniuc Simona-Ecaterina.

Motivation and Objectives

At the beginning of the semester we had to choose one project from the project proposals list that is available on the course page. After a discussion regarding our option we decided to go with the "Visualious" project as we find the requirements very interesting. The project description reads:

"Propose interesting means of complex visualization and faceted navigation concerning the information provided by various online sources of public datasets. Consider different interaction techniques/contexts concerning at least two classes of users, software and hardware platforms, displays, and devices."

We organized our work, as required in the project requirements page, and we created a blog where we can track the progress of the work.

The main idea of the project is to create an application which allows the users to visualize data in appealing way. The purpose of the project is not to provide a 100% working solution, but to apply "Human-Computer Interaction" principles in order to create an application which will suit the targeted user in every possible way.

Etymology and Branding

Name and Etymology

Our first idea for the name was “Visualicious” – Visual + Delicious. Because we like seeing data in appealing ways. But this name was taken by a graphic designer and we won’t like our identities to be mistaken.

The second idea was “Visualistique” – Visual + Majestique. Based on the same principle as “Visualicious”, but with a different adjective.

After founding mostly of our data that we are going to use, we agreed that our application should be based on data about food. We did a long research and debate about what we should finally name our application, and finally we decided to name it “FoodCharts”. We think that it emphasize very close the application domain and what it wants to accomplish.

Personas

Another requirement for this project is to do user research and create personas profiles for the possible users. We used the template that was provided on the course page. We reviewed what type of user could be interested in using our application, and decided that it should be a person who is interested about the nutritional values that the ingredients from the food contains. Possible users could be: cooks, students at faculties about chemistry and food research, or people who may have health problems and they want to follow a diet. We used the personas methodology and created profiles for Primary/Secondary/Negative users. Our final personas template is available on this page from our blog.

QOC

Also we need to create use the QOC (Questions, Options and Criteria) methodology for this project, where we asked questions regarding design key issues, we reviewed possible options to response to each question, and we set an evaluation criteria concerning every option. An example in our case would be: Can all users get the generated diagrams from our application? And the options and criterias are: Option 1: Yes. Criteria 1: Anyone that accesses the app and wants to use the generated diagram for the selected data can get the code for the diagram. Criteria 2: Anyone that access the app and wants to use the generated diagram can copy the image Option 2: No. They full list of QOCs are available on our blog.

Implementation

For the implementation of this project we decided to use a classical client/server architecture, where we use the server side to import the data from various sources like CSV and excel files that will be downloaded from different data providers, and then parse and process it into usable JSON format that will be send to our client application. Regarding the technology we used Symfony PHP framework for the server side, and on the front-end we developed out application with AngularJS.

User testing

We also did some research on user testing, and created a test scenario, in which we ask the evaluated user to do the following things: access the application, select a food category, apply filters and visualize/download data from diagram. Regarding the persons that will test the application, we asked some of our colleagues from work to help us. They were perfect fit, because they represent the primary persona type - technical users, which are interested in food and diet. The list of tasks that they have to accomplish are the following:

  • Access application and select category - 30 seconds;
  • Access filter menu and select filters (understand relation between selecting a filter and the showed result) - 45 seconds;
  • Apply filter to view the nutritional value for 3 favourite food types - 2 minutes;
  • Change diagram type to have a different visualisation for the data - 30 seconds
  • Download diagram and save it locally on the computer - 30 seconds
  • Access settings page and change font option - 20 seconds
  • Change color them to a dark one - 20 seconds

Our main objectives for user testing are:

  • Primary goal is to improve the usability of a product.
  • Identify how intuitive are the filtering controls, for a user that sees them first time.
  • Evaluate the positioning on page and the interaction that the user will have with the menus, filters and diagrams.
  • Observe user capacity to understand how filters work and the capacity un understand data from diagrams.
  • Observe user response to the color palette and scheme.
