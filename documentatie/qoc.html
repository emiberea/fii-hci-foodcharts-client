QOCs

  1. Does the user needs to register in order to navigate through our app?

    No. We are using public data sets, so anyone can access them.

  2. Can all users get the generated diagrams from our application?

    Option 1: Yes.

    Criteria 1: Anyone that accesses the app and wants to use the generated diagram for the selected data can get the code for the diagram.

    Criteria 2: Anyone that access the app and wants to use the generated diagram can copy the image

    Option 2: No.

    Criteria 1: The users cannot copy the diagrams from our application.

    Criteria 2: If a user wants a diagram, they should register to have access to it.

  3. Where should we possition the filters?

    Option 1: On the left side of the screen.

    Criteria 1: Users are already used to find the fillters on the left side of the sceen.

    Criteria 2: Easy access all the time to the filters.

    Option 2: On the top side, with a slide down effect.

    Criteria 1: There will be more space on the page.

    Criteria 2: It will implie more interaction with the user, when he wants to change the filters.

  4. When should we apply a selected option from the filter?

    Option 1: Immediatly, when is selected/deselected

    Criteria 1: Users want to filter immediatly, not to waste any time.

    Criteria 2: The user may press the option by mistake and the filer will apply.

    Criteria 3: While appling one filter, the user might press another option and the filter process should start again.

    Option 2:By using an apply button for the filters.

    Criteria 1: Users might want to apply multiple filters at once.

    Criteria 2: A filter won't be applied by mistake, just because a user pressed it.

    Criteria 3: Users might get frustrated with the application when they see that the filter doesn't apply immediatly

  5. What is the best way to show the data?

    Option 1: Show a list.

    Criteria 1:The users will be able to see a list with the results.

    Criteria 2:The users will not have any interaction with the list.

    Criteria 3: The users will not be impresses with the data and might not come back to our application

    Option 2: Use different types of charts.

    Criteria 1: Users will sea the data represented in charts.

    Criteria 2: Users can choose the type of chart that he wants.

    Criteria 3: The users will be impress with the representation of the data and he might come back to our application.

  6. Will the user be able to interact with the diagrams?

    Option 1: No

    Criteria 1: They can only change the filters to change the data from the diagrams.

    Criteria 2: They will not be able to change the colors.

    Option 2: Yes.

    Criteria 1: The user can change the color of the diagrams, so they can adjust them to their sites/presentations.

    Criteria 2: On the desktop version, users can hover to see more information.

    Criteria 3: On the mobile version, users can tap to see more information.

  7. What type of filters should we have?

    Option 1: Single option, for each type of filter.

    Criteria 1: The user might want to choose multiple options from a category

    Option 2: Multiple option for each type of filter.

    Criteria 1:The user might want to choose multiple options from a category
